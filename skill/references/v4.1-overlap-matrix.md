# D2/D10 重叠测试矩阵 — V4.1 (2026-05-16)

验证 D2(物化性质) 与 D10(ADMET/药化过滤) 是否存在重复惩罚。

## 测试序列

| 序列 | 特征 | D2 | D10 | 重叠? |
|---|---|---|---|---|
| CRGDKGPDC | 正常环肽 | 7.5 | 7.5 | ✅ 无 |
| CCCC | poly-Cys | 8.5 | 6.0 | ✅ 无 — D2未罚Cys,D10罚PAINS |
| FFFF | poly-Phe | 5.0 | 6.5 | ✅ 无 — D2罚疏水,D10罚PAINS,但不同维度 |
| CRRRRGDKPDC | poly-Arg | 6.5 | 5.0 | ✅ 无 — D2罚电荷失衡,D10罚Poly-Arg警报 |
| CCC | odd-Cys | 8.5 | 6.5 | ✅ 无 — D2未罚,D10罚unpaired Cys |
| AVILMFWYAVILMFWY | 全疏水 | ~3.0 | ~7.5 | ✅ 无 — D2重罚(疏水比例>60%),D10仅MW检查 |

## 判定标准

D2<5 **且** D10<5 → 真重叠 (需要调权重或去重)  
否则 → 独立惩罚维度

## 已知重叠: Cys重复惩罚 (已修复)

V4.1 初版中 D10 的 `score_medchem_filter()` 在两个位置对奇数 Cys 计数:
1. 新加的序列试探 Cys 检测 (line ~468)
2. 旧代码的 `# Unpaired Cys count` 块 (line ~484)

导致 `CCCC` 等序列在 D10 中被双重罚分。修复: 删旧块,保留新的序列试探版本。
